More About SteemBet.Asia gambling service

SteemBet.Asia is a provably fair, blockchain based dice gambling service.

How do we tell that you're not rigging the games?

The result is always taken from the block following your bet. Because it is calculated from the next block ID, it is impossible to guess or influence the result.

Whats the difference between you and original steemdice ?

Steemdice was the original project however the security issue existed for hosting it anywhere other than github or similar. To prove it's authenticity and validity it had to make do a public service to prove there is nothing to hide, a poor work-around to detect malicious activity. Github provides free hosting/shell tools for the dev community which is awesome, yet hosting a live service on github is limited in bandwidth and performance. That kind of hosting makes the bot slow and laggy when more people start playing on the bot. Our bot is an experiment in running blockchain game services on a dedicated server, with hardended security, designed with the resources required to keep up with several players without performance or lag issues â€“Just like a gaming server should. You can still pull our fork of code and access it on Guthub.

How to calculate the results for myself?

Convert to a decimal integer the last 5 characters of the hexadecimal block ID. If it is higher than 999,999 then use the ID of the next block. Example dd007 to decimal = 905,223. In this case < 999,999 so now apply a modulus of 10^4 and divide it by 100. You just type into Google 905223%(10000)/100 and the result will be 52.23

Why do you need to have my active private key?

SteemBet.Asia is not storing your login credentials, and by Steem API they are not leaving your browser either. Same as it is on SteemIt.com, for any Wallet Transfer with a memo, you need your active/wif key, otherwise transfer/bet can't be completed. For more info look at the question regarding "Ultra-decreet and paranoid" :)

It all sounds very risky?

Well, people are using original steemdice for a year now. Since we are official fork on github, to check and compare tree history between our projects, we strongly suggest that you visit us at github.

Are there other measures you take?

We host the application on our own highly secured servers, and we host applications for a living. You can rest assured they are highly secure. Remember, all code is open source and available on GitHub for you to check. If anyone tampered with our bot or server where it runs, they would have to be on our server not on the Steem network. We have monitoring in place to detect and alert us of any malicious activity immediately. We also do protect our hosting files for just this domain additionally with CrawlProtect tool.

How else can I play, if I want to be ultra-decreet and paranoid?

You can send your bets to our bot from the wallet directly, with the memo in following format: 
{"type":"higher/lower","number":50}
Example: 
{"type":"lower","number":55}
would be sending bet with amount you transfer for a win on lower than 55. In any case, if you send wrong memo with just one space extra, bot will return your funds as invalid bet.
